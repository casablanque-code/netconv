What you get
ZERO TRUST
No open ports by default
Every environment registers in Tollgate (identity-aware proxy, JWT auth) on provisioning. Developers access infra via HTTPS — never raw SSH from the internet.
GOVERNANCE
Policy enforced before apply
OPA policies run before every OpenTofu apply. Cost limits, security rules, mandatory tags — violations block provisioning with a clear error message.
MULTI-CLOUD
Switch cloud, keep interface
AWS, Yandex Cloud, Oracle Cloud, Hetzner. Same form, same outputs, same security baseline. Switch provider with a dropdown — zero code changes.
SELF-HOSTED
You own everything
No SaaS vendor. Runs on your GitHub and your cloud. Secrets never leave your perimeter. No monthly platform fee, no lock-in.
AUTO-LIFECYCLE
Resources expire automatically
Every resource has a mandatory expiry date. Auto-destroy on schedule, final backup before deletion. No zombie environments burning budget.
AUDIT
Immutable audit trail
Every event logged with identity, timestamp, cost estimate, and policy results. Stored as GitHub Actions artifacts — tamper-evident by design.
vs alternatives
Why not just Terraform + Ansible?
No self-service
Raw Terraform requires CLI, state management, and provider knowledge. Developers can't use it without training — someone always ends up being the bottleneck.
No governance
Nothing stops a developer from opening port 5432 to 0.0.0.0/0 or spinning a $500/mo instance. Policy is a README, not enforcement.
No lifecycle
Terraform doesn't know when a resource should die. Staging environments from 6 months ago accumulate silently and burn budget.
No Zero Trust
Terraform creates the VM. Access control, network isolation, and audit trail are entirely your problem to solve separately — every time.
